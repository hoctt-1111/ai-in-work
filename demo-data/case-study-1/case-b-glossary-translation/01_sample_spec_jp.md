# 出荷管理 基本設計書（抜粋）
# Shipment Management — Basic Design Document (Excerpt)

## 1. 概要

本機能は、圃場で収穫された農産物の出荷予定を管理し、出荷先との連携を効率化するためのモジュールである。

### 1.1 対象業務
- 出荷予定の登録・編集・削除
- 出荷実績の記録
- 出荷先マスタの管理
- 防除暦との連携（出荷前の農薬使用期限チェック）

## 2. 機能一覧

| No. | 機能名 | 概要 |
| --- | --- | --- |
| F-001 | 出荷予定登録 | 圃場ごとの出荷予定を登録する |
| F-002 | 出荷実績入力 | 実際の出荷数量・日時を記録する |
| F-003 | 防除暦チェック | 出荷予定日と農薬散布日の間隔を自動チェックする |
| F-004 | 出荷先管理 | 出荷先（市場・直売所・加工業者）の情報を管理する |
| F-005 | 出荷レポート | 期間別・圃場別の出荷実績レポートを出力する |

## 3. 画面設計

### 3.1 出荷予定一覧画面
- 検索条件：圃場名、出荷先、出荷予定日（From-To）、品目
- 一覧表示項目：出荷予定No.、圃場名、品目、出荷数量（予定）、出荷予定日、出荷先、ステータス
- ステータス：未出荷、出荷済、キャンセル

### 3.2 出荷予定登録画面
- 入力項目：
  - 圃場（プルダウン選択）
  - 品目（圃場に紐づく品目から選択）
  - 出荷予定数量（kg）
  - 出荷予定日
  - 出荷先（プルダウン選択）
  - 備考

### 3.3 防除暦チェック結果
- 出荷予定登録時、該当圃場の防除暦を自動参照
- 農薬散布日から出荷予定日までの日数が使用基準日数未満の場合 → 警告表示
- 警告メッセージ例：「防除暦の使用基準に基づき、○○の出荷予定日を再確認してください」

## 4. ビジネスルール

### BR-001: 出荷数量チェック
- 出荷予定数量は、該当圃場の推定収穫量を超えてはならない

### BR-002: 防除暦連携
- 農薬の使用基準日数（残留農薬基準）を満たさない出荷予定は登録不可
- ただし、管理者権限で強制登録可能（理由入力必須）

### BR-003: 出荷先制約
- 出荷先がJAの場合、出荷規格（サイズ・等級）の入力必須
- 直売所の場合、規格入力は任意
