## 1. 概要

### 1.1 目的
本システムは、農産物の在庫管理を効率化し、出荷計画と連携することで、
廃棄ロスの削減と出荷効率の向上を実現するためのWebアプリケーションである。

### 1.2 対象ユーザー
| ユーザー種別 | 人数（想定） | 主な操作 |
| --- | --- | --- |
| 圃場管理者 | 50名 | 収穫登録、在庫確認 |
| 倉庫担当者 | 20名 | 入庫・出庫、棚卸 |
| 出荷管理者 | 10名 | 出荷指示、出荷実績 |
| 経営管理者 | 5名 | レポート閲覧、ダッシュボード |
| システム管理者 | 2名 | マスタ管理、ユーザー管理 |

### 1.3 システム構成
- フロントエンド: React 18 + TypeScript
- バックエンド: Java 17 + Spring Boot 3
- データベース: PostgreSQL 15
- インフラ: AWS (ECS, RDS, S3)
- 認証: OAuth 2.0 + SAML (社内SSO連携)

## 3. 画面設計

### 3.1 ダッシュボード画面 (SCR-001)

**目的:** 経営管理者が在庫状況を一目で把握する

**表示項目:**
- 在庫数量サマリー（品目カテゴリ別）
- 賞味期限アラート（3日以内に期限切れ）
- 今週の出荷予定 vs 在庫
- 廃棄率推移グラフ（直近3ヶ月）
- 圃場別収穫進捗

**更新頻度:** リアルタイム（WebSocket）

---

### 3.2 収穫登録画面 (SCR-002)

**目的:** 圃場管理者が収穫情報を登録する

**入力項目:**
| No. | 項目名 | 型 | 必須 | 備考 |
| --- | --- | --- | --- | --- |
| 1 | 収穫日 | 日付 | ○ | デフォルト: 当日 |
| 2 | 圃場 | プルダウン | ○ | ログインユーザーの担当圃場のみ |
| 3 | 品目 | プルダウン | ○ | 圃場に紐づく品目 |
| 4 | 収穫数量 | 数値 | ○ | 単位: kg、小数点1桁 |
| 5 | 等級 | プルダウン | ○ | 秀・優・良・外 |
| 6 | 収穫担当者 | テキスト | - | 複数名入力可 |
| 7 | 備考 | テキストエリア | - | 天候・作業メモ等 |

**バリデーション:**
- 収穫数量は0.1〜9999.9の範囲
- 収穫日は当日〜過去7日以内
- 同一圃場・同一品目・同一日の重複登録は警告表示（上書き確認）

### 3.3 在庫照会画面 (SCR-003)

**目的:** 現在の在庫を多角的に確認する

**検索条件:**
| No. | 条件 | 型 | 備考 |
| --- | --- | --- | --- |
| 1 | 品目カテゴリ | プルダウン | 野菜・果物・穀物 |
| 2 | 品目 | プルダウン | カテゴリ連動 |
| 3 | 倉庫 | プルダウン | 複数選択可 |
| 4 | 等級 | チェックボックス | 複数選択可 |
| 5 | 賞味期限 | 日付範囲 | From-To |
| 6 | 在庫数量 | 数値範囲 | From-To |

**一覧表示項目:**

- 残日数が3日以内: 赤色表示
- 残日数が7日以内: 黄色表示

**ソート:** 各列ソート可、デフォルトは賞味期限昇順
**エクスポート:** CSV/Excel出力対応

### 3.4 入庫管理画面 (SCR-004)

**目的:** 収穫物を倉庫に入庫する

**入力項目:**
| No. | 項目名 | 型 | 必須 | 備考 |

| 2 | 倉庫 | プルダウン | ○ | |
| 3 | 品目 | プルダウン | ○ | |
| 4 | 等級 | プルダウン | ○ | |
| 5 | 入庫数量 | 数値 | ○ | 単位: kg |
| 6 | 賞味期限 | 日付 | ○ | 品目マスタのデフォルト日数から自動計算 |

**対象選択:**

| 3 | 出庫数量 | 数値 | ○ | 在庫数量以下 |
| 4 | 出荷伝票No. | テキスト | ○ | |
| 5 | 運送業者 | プルダウン | - | |
| 6 | 備考 | テキストエリア | - | |

---

### 3.6 賞味期限管理画面 (SCR-006)

**目的:** 期限切れリスクのある在庫を管理する

**アラートルール:**
| アラートレベル | 条件 | アクション |
| --- | --- | --- |
| 🔴 緊急 | 期限まで1日以内 | 自動メール通知 + ダッシュボード赤表示 |
| 🟡 警告 | 期限まで3日以内 | ダッシュボード黄表示 |
| 🟢 注意 | 期限まで7日以内 | 一覧で色付け |

**対応操作:**
- 値引き出荷（出荷先変更）
- 加工向け転用
- 廃棄登録

---

### 3.7 棚卸画面 (SCR-007)

**目的:** 定期棚卸の実施と差異管理

**棚卸フロー:**
1. 棚卸開始 → 在庫データ凍結（棚卸中は入出庫不可）
2. 実棚数量入力（倉庫・品目ごと）
3. 差異確認（システム在庫 vs 実棚）
4. 差異理由入力
5. 棚卸確定 → 在庫数量補正

**差異レポート出力項目:**
- 品目、倉庫、システム在庫、実棚数量、差異、差異率、理由

## 4. ビジネスルール

### BR-001: 在庫数量整合性
- 在庫数量 = 累計入庫数量 - 累計出庫数量 - 廃棄数量
- マイナス在庫は許可しない
- 棚卸による補正時は、補正履歴を記録する

### BR-002: FIFO原則
- 出庫時は先入先出（First In, First Out）を原則とする
- 賞味期限が近い在庫を優先的に出庫する
- ただし、ユーザーが手動で対象を変更可能（理由入力必須）

### BR-003: 賞味期限自動計算
- 入庫時に品目マスタの「標準保存日数」から賞味期限を自動計算
- 品目ごとの標準保存日数: 野菜(5-14日)、果物(7-30日)、穀物(180-365日)
- 手動変更可能（変更理由入力必須）

### BR-004: 倉庫容量管理
- 各倉庫に最大容量(kg)を設定
- 現在使用量 = 倉庫内在庫合計
- 使用率 = 現在使用量 / 最大容量 × 100

### BR-005: 出荷先制約
- JAグループへの出荷: 等級「秀」「優」のみ
- 直売所への出荷: 等級制限なし
- 加工業者への出荷: 等級「良」「外」可

### BR-006: 廃棄ルール
- 賞味期限超過の在庫は自動的に「要廃棄」ステータスに変更
- 廃棄には管理者承認が必要
- 廃棄理由の記録必須（期限切れ/品質不良/破損/その他）

## 5. 非機能要件

### 5.1 パフォーマンス
- 画面表示: 3秒以内
- 在庫検索: 2秒以内（10万件データ）
- レポート生成: 10秒以内
- 同時接続ユーザー: 100名

### 5.2 可用性
- 稼働率: 99.5%以上
- 計画メンテナンス: 月1回、深夜2:00-5:00
- バックアップ: 日次フルバックアップ + WALアーカイブ

### 5.3 セキュリティ
- 通信暗号化: TLS 1.3
- データ暗号化: AES-256（個人情報のみ）
- 認証: OAuth 2.0 + 社内SSO
- 権限管理: ロールベース（5段階）
- 監査ログ: 全操作記録、90日保持

## 6. 外部連携

### 6.1 気象データAPI連携
- 気象庁APIから気温・降水量を取得
- 賞味期限の動的調整に使用（高温時は期限短縮）

### 6.2 出荷管理システム連携
- 既存の出荷管理システム（SYS-BD-2024-001）とAPI連携
- 出庫データを出荷管理システムに自動送信
- 出荷実績データを受信して在庫に反映

### 6.3 会計システム連携
- 廃棄データを会計システムに月次送信
- CSV形式、毎月1日 AM8:00に自動生成
