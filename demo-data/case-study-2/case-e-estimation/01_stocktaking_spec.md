# 棚卸機能 詳細設計書
# Stocktaking Feature — Detailed Design Document

**機能ID:** INV-005
**版数:** 1.0
**作成日:** 2024/11/10

---

## 1. 概要
本機能は、倉庫内の農産物在庫について定期棚卸を実施し、
システム在庫と実在庫の差異を管理するための機能である。

## 2. 画面設計

### 2.1 棚卸開始画面 (SCR-STK-001)

**表示/入力項目:**
| No. | 項目名 | 型 | 必須 | 備考 |
| --- | --- | --- | --- | --- |
| 1 | 棚卸日 | 日付 | ○ | デフォルト: 当日 |
| 2 | 対象倉庫 | プルダウン | ○ | 複数選択可 |
| 3 | 棚卸担当者 | テキスト | ○ | 複数名入力可 |
| 4 | 備考 | テキストエリア | - | |

**処理:**
- 「棚卸開始」ボタン押下で在庫データを凍結
- 棚卸中は対象倉庫の入出庫を停止（他倉庫は影響なし）
- ステータス: 未実施 → 実施中 → 差異確認中 → 完了

---

### 2.2 実棚入力画面 (SCR-STK-002)

**表示項目:**
- 倉庫名、品目名、等級、システム在庫数量

**入力項目:**
| No. | 項目名 | 型 | 必須 | 備考 |
| --- | --- | --- | --- | --- |
| 1 | 実棚数量 | 数値 | ○ | 単位: kg |
| 2 | 差異理由 | プルダウン | △ | 差異がある場合必須 |
| 3 | 備考 | テキストエリア | - | |

**差異理由選択肢:**
- 計量誤差
- 品質劣化による減少
- 記録漏れ（入庫/出庫）
- 盗難・紛失
- その他（自由入力）

---

### 2.3 差異確認画面 (SCR-STK-003)

**表示項目:**
| 列 | 内容 |
| --- | --- |
| 品目 | 品目名 |
| 倉庫 | 倉庫名 |
| システム在庫 | 棚卸開始時の在庫数量 |
| 実棚数量 | 入力された実棚数量 |
| 差異 | 実棚 - システム (kg) |
| 差異率 | 差異 / システム在庫 × 100 (%) |
| 理由 | 差異理由 |

**差異ハイライト:**
- 差異率 ±5%以内: 正常（白）
- 差異率 ±5%超〜10%: 警告（黄）
- 差異率 ±10%超: 異常（赤）→ 管理者承認必須

---

## 3. ビジネスルール

### BR-STK-001: 棚卸中の入出庫制御
- 棚卸中は対象倉庫の入庫・出庫操作を停止する
- 入出庫画面で対象倉庫を選択した場合 → エラー表示

### BR-STK-002: 差異承認
- 差異率±10%超の品目は管理者承認が必要
- 承認なしでは棚卸完了不可

### BR-STK-003: 在庫補正
- 棚卸完了時、実棚数量でシステム在庫を上書き
- 補正履歴を記録（補正前、補正後、差異、理由、承認者）

### BR-STK-004: 棚卸頻度
- 月次棚卸: 毎月末
- 臨時棚卸: 管理者がいつでも実施可能

## 4. エラーメッセージ
| コード | 条件 | メッセージ |
| --- | --- | --- |
| E-STK-001 | 実棚数量未入力 | 「実棚数量を入力してください」 |
| E-STK-002 | 差異理由未入力 | 「差異がある場合、理由を入力してください」 |
| E-STK-003 | 入出庫停止中 | 「棚卸中のため、この倉庫の入出庫は停止しています」 |
| E-STK-004 | 承認なし完了不可 | 「差異率10%超の品目があります。管理者の承認が必要です」 |
