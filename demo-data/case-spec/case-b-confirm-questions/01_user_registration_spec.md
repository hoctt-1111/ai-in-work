# ユーザー登録機能 詳細設計書
# User Registration — Detailed Design Document

**機能ID:** USR-001
**版数:** 1.0
**作成日:** 2024/11/01

---

## 1. 概要
本機能は、農産物在庫管理システムに新規ユーザーを登録するための機能である。

## 2. 画面設計

### 2.1 ユーザー登録画面 (SCR-USR-001)

**入力項目:**
| No. | 項目名 | 型 | 必須 | バリデーション |
| --- | --- | --- | --- | --- |
| 1 | 氏名 | テキスト | ○ | 最大50文字 |
| 2 | メールアドレス | テキスト | ○ | メール形式チェック |
| 3 | パスワード | パスワード | ○ | |
| 4 | パスワード確認 | パスワード | ○ | パスワードと一致 |
| 5 | 所属部署 | プルダウン | ○ | マスタから選択 |
| 6 | ロール | プルダウン | ○ | 管理者のみ変更可 |
| 7 | 電話番号 | テキスト | - | |
| 8 | プロフィール画像 | ファイル | - | |

### 2.2 処理フロー
1. ユーザーが必要項目を入力
2. 「登録」ボタン押下
3. バリデーションチェック
4. 確認メール送信
5. ユーザーがメール内リンクをクリック
6. アカウント有効化
7. 登録完了画面表示

## 3. ビジネスルール

### BR-USR-001: メールアドレス
- 社内ドメイン（@company-a.co.jp）のみ登録可能

### BR-USR-002: ロール
- デフォルトロール: 「一般ユーザー」
- 管理者ロールの付与には、既存管理者の承認が必要

### BR-USR-003: 初回ログイン
- 初回ログイン時にパスワード変更を強制する

## 4. エラーメッセージ
| No. | 条件 | メッセージ |
| --- | --- | --- |
| E-001 | 必須項目未入力 | 「{項目名}を入力してください」 |
| E-002 | メール形式不正 | 「正しいメールアドレスを入力してください」 |
| E-003 | パスワード不一致 | 「パスワードが一致しません」 |
| E-004 | メール重複 | 「このメールアドレスは既に登録されています」 |
