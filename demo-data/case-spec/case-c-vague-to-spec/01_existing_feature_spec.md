# Spec chức năng cũ (Reference format) — 出荷予定登録

**機能ID:** SHP-001
**版数:** 1.2

---

## 1. 概要
本機能は、圃場で収穫された農産物の出荷予定を登録する機能である。

## 2. 画面設計

### 2.1 出荷予定登録画面 (SCR-SHP-001)

**入力項目:**
| No. | 項目名 | 型 | 必須 | バリデーション | 備考 |
| --- | --- | --- | --- | --- | --- |
| 1 | 出荷予定日 | 日付 | ○ | 当日以降 | カレンダー選択 |
| 2 | 圃場 | プルダウン | ○ | — | 担当圃場のみ |
| 3 | 品目 | プルダウン | ○ | — | 圃場連動 |
| 4 | 出荷数量 | 数値 | ○ | 0.1〜9999.9 | 単位: kg |
| 5 | 出荷先 | プルダウン | ○ | — | マスタ参照 |
| 6 | 備考 | テキストエリア | - | 最大500文字 | — |

### 2.2 処理フロー
1. ユーザーが入力項目を入力
2. 「登録」ボタン押下
3. 防除暦チェック実行
4. チェックOK → 登録完了、ステータス「未出荷」
5. チェックNG → 警告表示、管理者承認で強制登録可能

## 3. Acceptance Criteria

### AC-001: 正常登録
```
Given: 出荷管理者が出荷予定登録画面を表示している
When: すべての必須項目を入力し、「登録」ボタンを押す
Then: 出荷予定が「未出荷」ステータスで登録され、一覧画面に遷移する
```

### AC-002: 防除暦警告
```
Given: 選択した圃場の防除暦で、出荷予定日が使用基準日数内である
When: 「登録」ボタンを押す
Then: 警告メッセージ「防除暦の使用基準に基づき、出荷予定日を再確認してください」が表示される
```

### AC-003: 数量超過
```
Given: 入力した出荷数量が、該当圃場の推定収穫量を超えている
When: 「登録」ボタンを押す
Then: エラーメッセージ「出荷予定数量が推定収穫量を超えています」が表示され、登録できない
```

## 4. エラーメッセージ
| コード | 条件 | メッセージ |
| --- | --- | --- |
| E-SHP-001 | 必須項目未入力 | 「{項目名}を入力してください」 |
| E-SHP-002 | 数量範囲外 | 「出荷数量は0.1〜9999.9の範囲で入力してください」 |
| E-SHP-003 | 数量超過 | 「出荷予定数量が推定収穫量を超えています」 |
| W-SHP-001 | 防除暦警告 | 「防除暦の使用基準に基づき、出荷予定日を再確認してください」 |
